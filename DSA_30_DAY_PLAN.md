# 30-Day DSA Plan (C/C++)

Goal: Build strong problem-solving fundamentals in 30 days with consistent coding practice.
Daily time target: 2-3 hours.

## Rules
- Learn concept (20-30 min)
- Implement from scratch in C/C++ (30-45 min)
- Solve problems (60-90 min)
- Review mistakes (15 min)

Problem target: 2-4 problems/day.

## Week 1: Basics + Arrays + Strings
### Day 1
- Big-O basics: O(1), O(n), O(log n), O(n log n)
- Array traversal, min/max, sum
- Problems: linear search, largest element, second largest

### Day 2
- Sorting basics: bubble sort, selection sort, insertion sort
- Implement all three sorts manually
- Problems: sort array, check sorted, kth smallest (small input)

### Day 3
- Prefix sum and frequency array
- Problems: range sum query (static), count frequencies

### Day 4
- Two pointers
- Problems: remove duplicates (sorted), pair sum, reverse array/string

### Day 5
- Sliding window (fixed size)
- Problems: max sum subarray of size k, first negative in window

### Day 6
- Strings in C/C++: parsing, palindrome, anagram check
- Problems: reverse words, valid palindrome, frequency compare

### Day 7 (Revision + Test)
- Re-implement 3 algorithms from memory
- Solve 5 mixed easy problems in 90 minutes

## Week 2: Recursion + Linked List + Stack/Queue
### Day 8
- Recursion fundamentals: base case, stack frames
- Problems: factorial, fibonacci, sum of digits, reverse array

### Day 9
- Recursion patterns: include/exclude, subsequences
- Problems: generate subsequences, subset sum (small)

### Day 10
- Singly linked list: insert/delete/search
- Implement custom linked list

### Day 11
- Linked list interview patterns
- Problems: reverse list, middle node, detect cycle

### Day 12
- Stack and queue implementation
- Problems: valid parentheses, next greater element

### Day 13
- Monotonic stack + basic deque usage
- Problems: stock span, daily temperatures

### Day 14 (Revision + Test)
- 1 mock set: 2 recursion + 2 linked list + 2 stack/queue

## Week 3: Binary Search + Trees + Heap
### Day 15
- Binary search basics and edge cases
- Problems: lower bound, upper bound, first/last occurrence

### Day 16
- Binary search on answer
- Problems: sqrt(x), allocate pages, capacity to ship packages

### Day 17
- Binary tree traversals (recursive + iterative)
- Problems: preorder/inorder/postorder, level order

### Day 18
- Tree properties
- Problems: height, diameter, balanced tree check

### Day 19
- BST operations
- Problems: search/insert/delete in BST, validate BST

### Day 20
- Heap and priority queue
- Problems: kth largest, merge k sorted arrays/lists

### Day 21 (Revision + Test)
- Rebuild tree + heap templates from scratch
- 5 mixed medium problems

## Week 4: Graph + DP + Interview Practice
### Day 22
- Graph representation, BFS, DFS
- Problems: connected components, number of islands

### Day 23
- Topological sort + cycle detection
- Problems: course schedule, detect cycle in directed graph

### Day 24
- Shortest path basics
- Problems: BFS shortest path (unweighted), Dijkstra intro

### Day 25
- DP intro: memoization vs tabulation
- Problems: climbing stairs, house robber, frog jump

### Day 26
- Classic DP
- Problems: 0/1 knapsack, coin change, LIS

### Day 27
- Strings + DP
- Problems: LCS, edit distance, longest palindromic subsequence

### Day 28
- Backtracking
- Problems: N-Queens, permutations, combination sum

### Day 29
- Full mock interview round
- 2 easy + 2 medium + 1 hard (timed)

### Day 30 (Final Review)
- Analyze all mistakes from the month
- Create your personal "revision sheet"
- Re-solve 10 previously wrong problems

## Recommended Pattern for Every Problem
1. Brute force idea
2. Better approach
3. Optimal approach
4. Complexity analysis
5. Clean final implementation

## Weekly Outcome Checklist
- Week 1: Comfortable with arrays/strings patterns
- Week 2: Strong recursion and linear data structures
- Week 3: Binary search + trees + heap confidence
- Week 4: Graph + DP + interview readiness

## If You Have Less Time (1 hour/day)
- 15 min concept
- 15 min implementation
- 30 min one quality problem
